- Branches and calls are emitted as provisional forms whose final encoding can depend on link-time distance, target address, and architecture-specific range limits.
- Patchable branch and call sites are represented explicitly, with fixed-width or metadata-backed forms used where later mutation needs atomicity or stable offsets.
- Repatching is separate from first-time linking: per-architecture helpers know which instruction word or pointer load actually carries the mutable target.
- Link-time relocation handles constant-pool flushes, MIPS/ARM jump relocation, and jump islands before executable code is made visible.

## Facts

- 2010-05-08 (98d62906) pitfall: Cortex-A8 can misread or deadlock on a 32-bit Thumb-2 branch spanning a 4 KiB page boundary with a backward target, so those cases force absolute non-relative encoding. (sourced)
- 2013-10-22 (e0c6b73d) pitfall: LinkBuffer must force AssemblerBufferWithConstantPool to flush pending constants before measuring code size and allocating linked code. (code)
- 2013-11-05 (3df2f9af) pitfall: MIPS code copied by LinkBuffer must run MIPSAssembler::relocateJumps after memcpy because MIPS jump encodings are adjusted from the old buffer base to the final code base. (code)
- 2018-04-05 (d3e64bda) rationale: MIPS patchable branch padding belongs in branchPtrWithPatch rather than every branchEqual/branchNotEqual because the two NOPs only protect code that may be replaced by and reverted from a four-instruction jump. (sourced)
- 2024-05-03 (e9b55681) pitfall: Thumb2 concurrent call repatching cannot safely use the old mov-plus-blx shape; mov needs an ISB, and a 32-bit bl/b patch must be 4-byte aligned for one aligned-word update. (sourced)

## Moves

- 2009-07-22 (2e878a27) replaced [[link-repatch-buffer-nested-in-abstract-macro-assembler]]: Moving LinkBuffer and RepatchBuffer out of AbstractMacroAssembler into standalone headers was the first step in enabling per-architecture linkCall/repatchCall implementations that cannot live inside the shared base class. (sourced)
- 2009-07-22 (fc2a660c) replaced [[call-link-repatch-in-abstract-macro-assembler]]: x86-64 far calls are implemented as load-to-r11 then call-r11, so repatchCall must patch the pointer load at offset -3 rather than the call itself — this is an arch-specific invariant that cannot be expressed in the shared AbstractMacroAssembler. (sourced)
- 2009-10-29 (e27b33be) replaced [[armv7-relative-jump-link]]: The old linkWithOffset computed a relative offset at link time and encoded it directly into the branch instruction immediately; this failed when the offset exceeded the 16 MB range of Thumb-2 B.W, so the new scheme pre-plans all jumps as MOV/MOVT+BX sequences (absolute), and defers final encoding to executableCopy() where the actual target address is known, falling back to a relative B.W only if the distance fits. (sourced)
- 2010-10-05 (c24ff4ce) replaced [[armv7-two-level-branch]]: The old scheme had only ShortJump (T4, 4-byte) and LongJump (BX, 10-byte); adding T2 (2-byte unconditional branch, OP_B_T2) gives a third form the old enum could not represent, reducing JIT code size when the target is within ±2KB. (code)
- 2010-10-14 (93647b9f) replaced [[armv7-jump-single-padding-size]]: Conditional and unconditional jumps require different maximum padding sizes (12 vs 10 bytes), making a single JumpPaddingSize constant too large for conditionals; per-type padding allows the IT instruction to be included in the conditional sequence, enabling B(T1)/B(T3) short conditional branch encodings that shrink code size. (sourced)
- 2011-04-30 (b495c300) dropped: LinkBuffer recompile offset: The linkOffset parameter allowed LinkBuffer to resolve relative branches as if code resided at a different base address, enabling recompilation purely to regenerate exception-handler info without relinking; once the exception-info recompilation feature was removed from JSC, the parameter became dead and was eliminated. (sourced)
- 2011-05-02 (02f3ae07) replaced [[assembler-label-as-unsigned-int]]: The old unsigned int return type from label() was implicitly convertible to/from raw integers, allowing accidental arithmetic on offsets and making jump lists store raw ints; AssemblerLabel provides a type wall so consumers cannot silently mix offset values with unrelated integers. (sourced)
- 2011-07-07 (61a9b3cb) replaced [[armv7-jump-size-out-of-line-arrays]]: Static out-of-line arrays required a memory load and defeated constant folding of jump-delta arithmetic; encoding sizes directly into the enum values as high bits (JUMP_ENUM_WITH_SIZE macro) lets the compiler constant-fold all linking arithmetic and eliminates the array loads. (sourced)
- 2012-04-25 (a2d9e4a5) replaced [[uninterrupted-sequence-patchability-marker]]: Patchability became explicit in the branch/jump type instead of being inferred from uninterrupted-sequence state, so only patchable branches can have locations taken and ARMv7 fixed-width jumps are requested directly at emission sites. (code)
- 2014-05-14 (626b6a5c) replaced [[bare-jit-breakpoint-assertions]]: Coded abort reasons make a JIT SIGTRAP diagnoseable from the platform abort-reason register instead of from only the trap address. (sourced)
- 2017-06-24 (df1555bc) replaced [[vmtrap-breakpoint-instruction]]: Using breakpoint instructions for VMTraps conflicted with lldb, while VMTraps only required an exceptioning instruction to stop execution. (sourced)
- 2021-08-20 (21ea32f9) removed: x86-32 MacroAssembler backend: MacroAssemblerX86 was deleted, eliminating the CPU(X86) MacroAssembler target selection and leaving x86-family support through MacroAssemblerX86Common and MacroAssemblerX86_64 only. (code)
