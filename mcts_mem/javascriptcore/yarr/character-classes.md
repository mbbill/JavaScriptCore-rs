- Character classes are compacted into tables, sorted ranges, and immediate bit-vector tests rather than always emitted as branch trees.
- Unicode canonicalization is table-driven and split between legacy UCS2 and Unicode case-folding modes.
- Character-class storage and code generation track subject-width boundaries including Latin1/non-Latin1 and BMP/non-BMP distinctions.
- Named and numbered backreferences are separate term types; duplicate-name indirection does not change numeric reference targets.

## Facts

- 2009-05-01 (8f8daa0d) pitfall: When the upper bound of a case-insensitive character class range is 0xFFFF, the loop variable unicodeCurr (declared as UChar, 16-bit) overflows to 0x0000 on increment, causing an infinite loop; fix promotes unicodeCurr to uint32_t so it can reach 0x10000 and fail the loop condition. (code)
- 2010-12-09 (ef87ea04) rationale: Yarr regex parser previously rejected \d and other built-in class escapes inside character ranges (e.g., /[\d-x]/) as syntax errors per strict ECMA-262 conformance, but this was reverted because it broke real-world web content; the chosen policy is to accept such ranges silently (treating the hyphen as escaped), while still refusing to parse the trailing part as a range when the grammar would not permit it (e.g., /[\d-a-z]/ does not treat a-z as a range). (sourced)
- 2016-03-04 (bb6eb2e6) pitfall: Unicode RegExp syntax checking must receive the flags and reject IdentityEscape forms that are only valid without the u flag, while still allowing the class-only Unicode escape \-. (code)
- 2017-08-24 (163dde9c) rationale: Yarr represents the parsed '.' token as DotClassID and resolves it during pattern construction to either anyCharacterClass for dotAll or inverted newlineCharacterClass otherwise, keeping the JIT and interpreter on ordinary character-class terms. (code)
- 2019-04-03 (ed8966d9) pitfall: Greedy counted character class JIT loop for non-BMP-only character classes used atEndOfInput() (checks index != length, i.e. at least 1 char left) but non-BMP characters consume 2 code units, so the loop could advance index past end; fix adds checkNotEnoughInput(1) (checks index+1 < length) for non-BMP-only classes without inversion. (code)
- 2024-03-13 (67969c21) rationale: The JIT avoids checking the error code point for normal non-inverted atoms because that sentinel cannot match them, but inverted atoms must explicitly fail on the sentinel. (sourced)
- 2024-05-23 (a8e4fcd2) pitfall: The ARM64 Yarr optimization that advanced a failed match by an extra code unit after reading a non-BMP surrogate pair was removed after a site regression; a regexp restart point cannot be advanced by surrogate-pair width merely because the first read decoded a non-BMP code point. (sourced)
- 2024-08-23 (0f30a2dc) measurement: The relanded character-class matcher reported about 10% improvement on an ASCII alphanumeric microbenchmark, about 20% on a single-chars microbenchmark, and about 2% on a separator-chars microbenchmark. (sourced)
- 2024-08-23 (0f30a2dc) pitfall: When partitioning matches around a selected inclusive range, higherMatchStart must advance while match <= hi; using < treats a literal equal to the range end as a higher unmatched literal instead of covered by the current range. (sourced)
- 2025-01-14 (1ab331e3) pitfall: Yarr bytecode compilation must check overflow on every derived input offset, including currentCountAlreadyChecked plus or minus term.inputPosition, and return OffsetTooLarge rather than emitting bytecode with wrapped offsets. (code)
- 2026-05-07 (0aef258b) measurement: Encoding Unicode property string tables as flat constexpr char32_t data plus length arrays and making CharacterRange constexpr reduced generated \p{...} construction code from about 468 KB to 63 KB, moved about 150 KB to __TEXT,__const, and netted about 283 KB binary-size reduction without RegExp matching slowdown. (sourced)

## Moves

- 2006-11-20 (090b9a95) replaced [[regexp-unicode-escape-in-lexer]]: Handling \u escapes in the lexer (r17354) broke metacharacter escaping and prevented serialized regexps from preserving unicode escapes; moving translation to the RegExp constructor via sanitizePattern() better matches other browsers' behavior by keeping the raw pattern string intact through the parser. (sourced)
- 2010-04-20 (7eaea7d0) replaced [[yarr-builtin-charclass-range-check]]: Hand-written range/match vector constructors and binary-search JIT dispatch replaced by Python-generated 256-byte bitmap tables and a single branchTest8 memory read, enabling O(1) character class membership tests for ASCII built-ins. (code)
- 2012-03-26 (d881db51) replaced [[yarr-pairwise-unicode-case-folding]]: The old mechanism assumed each codepoint has at most one additional case-insensitive match obtainable by Unicode::toUpper/Unicode::toLower, while the new mechanism can represent ES5.1 canonical equivalence sets and range encodings. (sourced)
- 2013-04-12 (91d2eafc) replaced [[character-class-table-separate-allocation]]: CharacterClassTable was a RefCounted heap object holding only a const char* pointer and a bool, making a separate allocation wasteful for what could be two inline fields directly in CharacterClass. (code)
- 2016-03-08 (1d5ebf95) replaced [[yarr-unified-generated-unicode-canonicalization]]: Unicode regexp canonicalization moved from a checked-in unified table to separate legacy UCS2 tables plus a build-generated Unicode table from Unicode CaseFolding.txt so Unicode-mode matching follows ES6 Canonicalize data independently of non-Unicode behavior. (sourced)
- 2019-03-29 (71a9b2c0) replaced [[yarr-charclass-bmp-bool-flag]]: A single bool could not distinguish BMP-only, non-BMP-only, and mixed character classes, so the JIT always emitted the expensive generic isBMP branch; replacing with a 2-bit enum enables fixed-width advance code paths for pure-BMP and pure-non-BMP classes, eliminating the branch entirely. (code)
- 2024-06-27 (34b0b047) replaced [[yarr-character-class-branch-search]]: Dense nearby character-class members are tested by subtracting the minimum and masking against an immediate bit vector instead of emitting O(n) equality branches or an all-or-nothing range binary search. (code)
- 2024-07-27 (63dab201) replaced [[yarr-character-class-bitset-jit]]: The bitset optimization path was reverted after matchCharacterClassSet sometimes crashed when using it. (sourced)
- 2024-08-23 (0f30a2dc) replaced [[yarr-character-class-branch-tree-matching]]: YarrJIT replaced range/match branch trees and ASCII-letter special casing with recursive grouping plus biased immediate bitset tests because clustered character classes can be matched with subtract/shift/branchTest instead of O(n) equality branches. (sourced)
- 2025-08-18 (d96ab2fa) replaced [[yarr-unicode-first-read-sentinel]]: Sentinel-based first-read tracking could silently fail across skipped alternatives or backtracking control-flow changes, so each alternative now initializes the additional read size to 0 and non-BMP reads write 1 directly. (code)
- 2026-01-10 (75ec3982) replaced [[yarr-interpreter-vector-character-class-lookup]]: The Yarr interpreter now uses CharacterClass::m_table for characters below CharacterClass::tableSize before falling back to vector match/range searches, adopting the same table representation already used by the JIT. (code)
- 2026-03-20 (12ef604e) replaced [[yarr-untyped-backreference-term]]: Yarr split named and numbered backreference term types because duplicate-named-capture indirection is only correct for named references, while numeric references must keep targeting the explicitly numbered capture. (code)
- 2026-06-01 (bd24d557) replaced [[yarr-ascii-unicode-character-class-partition]]: Yarr character classes changed their storage split from ASCII/non-ASCII to Latin1/non-Latin1 to match the engine's Char8/Char16 execution boundary and avoid copying/clipping Unicode-side data for 8-bit input. (code)
