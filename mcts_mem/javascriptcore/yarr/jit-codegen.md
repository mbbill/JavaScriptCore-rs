- YARR JIT generation separates forward matching from reverse-order backtracking/continuation code instead of interleaving every path inline.
- Generated regexp code uses explicit register, ABI, and stack-limit/matching-context contracts at its entry boundary.
- Quantified parentheses and backtracking state are modeled explicitly enough to be code-generated when safe and routed conservatively otherwise.
- Anchored literal string alternations and bulk character comparisons get specialized generated code instead of generic per-term backtracking.

## Facts

- 2009-04-28 (463dacc4) rationale: YARR introduced a regexp-specific JIT path alongside interpreter bytecode rather than extending WREC's single JIT-only execution model. (code)
- 2010-05-20 rationale: Continuation labels and backtracking records make generated regexp code preserve the same control-flow choices as interpreter bytecode while optimizing the common fall-through path. (code)
- 2012-06-14 pitfall: Generated regexp entry code must preserve the register contract expected by both JIT entry stubs and standalone RegExp execution. (code)
- 2014-05-23 (836527ff) measurement: Avoiding repeated regexp compilation and output-vector materialization was a recurring source of SunSpider and Dromaeo wins, motivating separate match-only and capture-producing code paths. (sourced)
- 2020-11-04 (b2032c2e) rationale: The YARR JIT's register allocation is fixed by generated-code ABI contracts, so helper calls and inline thunks must remap registers explicitly. (code)
- 2020-11-04 (b2032c2e) pitfall: Backtracking over Unicode lookbehind cannot replay long prefixes repeatedly without turning a local backtrack into repeated rescans. (code)
- 2026-02-03 (eef70034) rationale: FixedCount ParenContext saves both begin and end indexes at ParenthesesSubpatternEnd because alternative retry needs the iteration begin index while quantified-content backtracking needs the iteration end index. (code)
- 2026-02-03 (eef70034) pitfall: Failed later fixed-count iterations can allocate Begin-created contexts that never reach End; backtracking must mark those contexts incomplete with matchAmount == -1 and skip them. (code)

## Moves

- 2010-11-29 (a4df4828) replaced [[jit-only-terminal-paren-optimization]]: Moving isTerminal detection from the JIT into the compiler's checkForTerminalParentheses() pass allows the interpreter to also exploit the no-backtrack optimization for greedy unbounded non-capturing parentheses at the end of a regex, and is required because frame-size allocation (which must account for terminal paren frame slots) happens at compile time, not JIT time. (code)
- 2011-05-16 (8f351921) replaced [[yarr-jit-mixed-pass-codegen]]: The old single-pass generator interleaved forward-match and backtrack code inline, requiring IndirectJumpHashMap and ParenthesesTail callbacks for out-of-line backtrack emission; the new approach first compiles the pattern to a linear YarrOp sequence, then emits all forward code in one pass and all backtrack code in reverse order (so the common fall-through is optimized) in a separate pass via BacktrackingState. (code)
- 2012-03-28 (f795f157) replaced [[regexp-single-output-vector-match]]: The old API always allocated and populated an output vector even when callers only needed the match span, while the new API can compile and cache a JIT variant that omits subpattern output writes. (sourced)
- 2012-05-01 (5c6e9dc3) replaced [[single-jit-availability-gate]]: Need to split canUseRegExpJIT out of canUseJIT. (sourced)
- 2016-07-14 (418d7655) replaced [[yarr-signed-subject-offsets]]: YARR switched from signed relative offsets to Checked<unsigned> negative-character distances because large quantified regular expressions can exceed signed displacement ranges, and the JIT now biases the base register when BaseIndex cannot encode the negative offset. (code)
- 2018-01-29 (88fdfc6a) replaced [[regexp-jit-vm-execution-flag]]: A VM boolean toggled inside generated Yarr code could be sampled before it was set or after it was cleared and forced YarrJIT to depend on VM, so the replacement marks the exact RegExp object with a main-thread RAII tracer around JIT execution and lets the profiler emit RegExp frames. (code)
- 2018-08-21 (4ce019d1) replaced [[yarr-jit-bulk-match-32bit]]: On 64-bit platforms (X86_64, ARM64) a single GPR can hold 8 bytes; extending the character accumulator from int/uint32_t to uint64_t and adding a check8 lambda (load64/branch64) allows matching up to 8 characters in one compare, reducing code size by fusing multiple mov+cmp pairs into one. (code)
- 2021-11-11 (b5429a89) replaced [[yarr-standalone-regexp-test-callout]]: RegExp.test gained an inline DFG/FTL path by making Yarr emit into a caller-provided MacroAssembler with caller-selected registers, avoiding the C++ operation call for eligible 8-bit non-rope non-global/sticky non-Unicode cases. (code)
- 2023-06-28 (8e5b3f57) replaced [[regexp-lookbehind-rematch-backtracking]]: Unicode lookbehind character-class backtracking now unreads one codepoint instead of rematching one fewer codepoint, avoiding repeated rescans when many codepoints must be backed off. (code)
- 2024-06-26 (778df0f6) replaced [[windows-abi-yarr-jit]]: Windows Yarr JIT now uses SYSV_ABI and the generic x86_64 register contract, allowing RegExp JIT to remain enabled on Windows instead of carrying a separate hidden-return-pointer ABI path. (code)
- 2025-02-21 (12c34ef5) replaced [[yarr-nested-alternative-backtracking-for-string-lists]]: Anchored non-capturing alternations of literal strings were recognized as non-backtracking string lists so the JIT could jump on a whole-alternative match instead of emitting per-term fall-through and backtracking continuation code. (code)
- 2025-02-24 (5d4feeae) replaced [[yarr-backtracking-string-alternation]]: Anchored non-capturing alternations of fixed strings can stop at the first matched string and jump on match, eliminating the usual continuation-PC backtracking path. (code)
- 2026-02-04 (4a3cd3e0) replaced [[fixedcount-parentheses-special-case-backtracking]]: FixedCount parentheses stopped distinguishing backtrackable from non-backtrackable cases and conservatively routes FixedCount groups through ParenContext/content-backtracking because the previous matchAmount decrement and m_reentry special cases were too error-prone. (code)
- 2026-04-23 (282d55d7) replaced [[yarr-fixedcount-snapshot-clearing-backtracking]]: FixedCount ParenContext backtracking changed from restoring snapshots then clearing potentially stale parenContextHead slots to marking the current context incomplete and reusing or freeing incomplete contexts, which bounds FixedCount{N} to at most N ParenContext allocations without restoring stale freed heads. (code)
- 2026-06-02 (6646c49f) replaced [[yarr-expanded-min-plus-variable-parentheses]]: Forward-direction parenthesized subpatterns with non-zero minimum are kept as one quantified term because expanding them into FixedCount{min} plus variable tail deep-copied the disjunction subtree and could hit OffsetTooLarge or pattern-size limits for very large bounds. (code)
- 2026-06-04 (a92d79b2) replaced [[yarr-fixedcount-save-at-end]]: FixedCount switched from a special save-at-END ParenContext model to the same save-at-BEGIN snapshot model as Greedy and NonGreedy so quantified-parentheses backtracking could share one state machine. (code)
