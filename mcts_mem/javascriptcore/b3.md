- B3 is a language-neutral SSA backend lowered to Air before final allocation and emission. (`B3::Procedure`)
- B3 operation identity is a Kind: an opcode plus independent flags for chill behavior, trapping memory, and related mode bits.
- Basic blocks own terminal successor edges; terminal values can describe generated control flow.
- Procedure state is split at lowering time: Air keeps the lowered code, selected origin data, and stackmap-visible values, while most B3 graph state is released before machine-code generation.
- B3 value children use shape-specific inline storage for common fixed arities and varargs storage only when needed.
- Air is a reflective near-machine IR whose opcode forms advertise target availability and effects to selection, DCE, allocation, and generation.
- Air block ordering is CFG-shape-aware, separating fallthrough-chain continuation from restart choices.

## Facts

- 2015-10-28 (a6816fc4) rationale: B3 was introduced as a low-level SSA-based, language-agnostic compiler for aggressive C-level optimizations and a portable backend (sourced).
- 2015-10-28 (a6816fc4) rationale: Air's opcode metadata lets instruction selection query declared valid forms instead of hard-coding one lowering shape for every target form (code).
- 2016-07-19 (53a1e5c7) statement: LLVM-style indirectbr/blockaddress would tie successor edges to exact addresses, preventing B3 from splitting edges for SSA and making those branches non-cloneable for specialization transforms (sourced).
- 2016-09-29 (40a45a42) pitfall: phases that transcribe B3 IR can silently drop Kind bits if they copy only the opcode rather than the full operation identity (code).
- 2017-08-12 (753a4af6) measurement: changing caged adds from patchpoints to Opaque was reported performance-neutral, while removing the reassociation barrier made Octane 1.6% slower and Kraken 5% slower (sourced).
- 2019-04-15 (402606bd) measurement: tagged inline child storage halved memory for zero-child Values and reduced Add from 72 to 48 bytes (sourced).
- 2019-04-15 (402606bd) statement: zero-child Values still allocate one pointer-sized child slot for replaceWithIdentity to store a child in place (code).

## Moves

- 2015-11-18 (257cf285) replaced [[b3-monolithic-generate-pipeline]]: The B3 pipeline was split so expensive B3 and Air preparation can run inside the FTL graph safepoint while final machine-code generation and stackmap callbacks run after leaving it. (sourced)
- 2016-07-19 (53a1e5c7) replaced [[b3-controlvalue-owned-successors]]: Successor edges moved from ControlValue into BasicBlock so any terminal Value, especially a Patchpoint with effects.terminal, can own arbitrary generated control flow without imposing a special ControlValue subclass. (sourced)
- 2016-09-29 (40a45a42) replaced [[b3-opcode-only-value-kind]]: B3 needed multidimensional operation identity because a one-dimensional Opcode enum would require combinatorial opcode growth for independent flags such as chillness, trapping loads, and memory-ordering modes, while subclass fields would not participate in Value::accepts and ValueKey by default. (sourced)
- 2016-10-01 (c1aed292) replaced [[single-extra-bit-b3-kind]]: A single chill extra bit could not represent memory accesses whose traps are observable, so B3::Kind became a bag of flags with a traps bit that affects effects analysis, store elimination, validation, and Air lowering. (code)
- 2017-08-12 (753a4af6) replaced [[patchpoint-hidden-caged-add]]: The patchpoint form hid Add(ptr, largeConstant) from harmful B3 reassociation but also imposed patchpoint costs, backend callbacks, and prevented CSE, while Opaque blocks pre-Air reasoning yet remains pure, idempotent, and CSE-able until LowerToAir treats it as Identity. (code)
- 2019-04-15 (402606bd) replaced [[b3-value-uniform-children-vector]]: The fixed Vector<Value*, 3> m_children (40 bytes) allocated the same footprint regardless of how many children a Value actually used; replacing it with a kind-tagged inline storage (0/1/2/3 pointers allocated immediately after the object, or a VarArgs Vector) halves memory for zero-child Values such as Const64/Const32/Nop/Identity and reduces Add from 72 to 48 bytes. (sourced)
- 2025-11-11 (01f76792) replaced [[air-frequency-stack-block-order]]: The old ordering used one LIFO worklist for both fallthrough-chain continuation and broken-chain restart, while the new ordering separates those choices and injects explicit triangle, diamond, and exclusive-successor CFG layouts. (code)
